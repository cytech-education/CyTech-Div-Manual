# アカウント発行・管理

CyTech 各種アカウントの発行フローと、発行後の管理方法をまとめています。

## アカウント発行方法

### 社内アカウント

- **発行者**：営業担当
- 手順：
  1. 営業担当がアカウントを発行
  2. 発行後、**フォローアップシート** に情報を追記

### ToC アカウント

発行依頼フロー：

1. カウンセラーから GL（紀谷）に発行依頼
2. GL が各 IR（インストラクター）に発行を依頼
3. 基本的に発行を行った**各 IR が担当**

アカウント作成会社：**株式会社 TheNewGateToC**

:::danger 管理者アカウント情報
ToC 管理者アカウントのメールアドレス・パスワードは**マニュアル外で別管理**しています。
必要な場合は GL（紀谷）までご連絡ください。
（※ Git 管理される本マニュアルに直書きしない運用です）
:::

発行後の作業：

- **カウンセリングシート**に以下を追記
  - 開始日
  - ユーザー ID
  - 念の為、パスワード
- **初回面談の案内メール**を送付（[入学〜初回MTG](../getting-started/admission) 参照）

### ToB アカウント

- 発行方法：**現時点では未定**（今後決定予定）

## アカウント管理方法（新 CyTech）

::: info
旧 CyTech については**従来通り**の運用を継続します。
:::

### 運用手順

1. 営業が対象ユーザーにアカウントを付与
2. **フォローアップシート（新 CyTech）** に付与情報を記入
3. 入力された情報は、自動的に評価基準シート「**新 Engineer 管理**」に反映される
4. 今後の管理は「評価基準シート」で**一括管理**を行う（※ 新 CyTech のみ）
5. 評価基準シート内では、以下を入力：
   - 提出日
   - 回数
   - 合格日（※ 合格日に入力するとセルの色が変化し、完了状況が視覚的に把握可能）
6. **F 列**の枠には、**担当インストラクターの名前を必ず入力**すること

## 全体的な管理

- **管理スプレッドシート**：Google スプレッドシート（全体管理） — [リンク集](../links)参照
- **定期作業**：
  - データベースから追加ユーザーを定期的に抽出し、上記スプレッドシートへ**手動で追加**
  - 追加後、**担当割り振り**を行う

### 追加ユーザー抽出用 SQL

以下の SQL をデータベースに対して実行し、新規ユーザーを抽出する。`user_id` は
シートに記載されている ID 以降で調整する。

```sql
SELECT
  u.user_id,
  CONCAT(u.last_name, u.first_name) AS fullName,
  CONCAT(u.last_name_kana, u.first_name_kana) AS kana,
  u.email_address,
  u.created_at
FROM
  users u
JOIN
  user_permissions up
ON
  u.user_id = up.user_id
-- user_id はシートに記載されている ID 以降に調整
WHERE
  u.user_id >= 172
  AND (up.permission_id = 8 OR up.permission_id = 9)
ORDER BY
  u.user_id ASC;
```
