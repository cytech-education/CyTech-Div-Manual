# チーム開発（後半カリキュラム）の運用マニュアル

## 全体フロー
1. STEP9終了
2. リポジトリを用意
3. 各種ファイルを用意
4. 説明面談を実施
5. ＳＴＡＲＴ
6. 各ステップレビュー実施
7. 結合テストまで終了
9. スキルシート作成面談実施
10. END

### フロー
1. 管理フォルダに、ナンバリングでチームフォルダを作成
2. チームフォルダに、テンプレートからのコピーで「各種情報」ファイルと、「WBS」ファイルを作成
3. 各種仕様書提出フォルダを作成
4. 各種情報ファイルの仕様書、ＷＢＳのリンクを差し替え
5. リポジトリ、ブランチ、開始日（説明MTG日）、担当IR、メンバーを記入
6. 管理フォルダの編集権限を対象ユーザーに付与


## 説明面談について
### 話す内容
1. 後半カリキュラムの概要
    1.1. コーディング以外の開発業務（設計やテスト）も経験してもらう
    1.2. ～STEP12は、すでにあるページの改修、STEP13～新規ページの実装
    1.3. 今回触ってもらうのはCyTechの管理画面であること
2. リポジトリの共有について
    1.1. 共有するリポジトリを落としてもらって、READMEを参考に初期設定→立ち上げまでをまずやってもらう
3. 各種情報ファイル
    1.1. 課題（問題集）からやってもらう課題を説明（一人の場合Companyをやってもらってます）
    1.2. 課題、レビューのフローの説明（基本的には課題全て実装してレビューではなく、課題毎にレビュー。STEP16結合テストは除く）
    1.3. 提出はChatから
    1.4. WBSの説明（どういうものか、期限の目安（３ヶ月））
